Middleware Wrapper Pattern

Bickford sits between enterprise applications and Claude API, adding enforcement without modifying foundation models.

// Standard Claude API Call (Before Bickford) const response = await claude.complete({ prompt: "Analyze patient symptoms...", model: "claude-sonnet-4-5" }); // Bickford-Enhanced Claude Call (After Integration) const response = await bickford.enforced_complete({ prompt: "Analyze patient symptoms...", model: "claude-sonnet-4-5", enforcement_policy: "HIPAA_HEALTHCARE", audit_context: { user_role: "physician", patient_id: "***REDACTED***", clearance_level: "PHI_AUTHORIZED" } }); // Response includes cryptographic proof // response.audit_hash: "4a3f9c2b1e...d8f6" // response.enforcement_proof: { constraints_applied: [...] }
Zero-Touch Foundation Models: Bickford never modifies Claude's weights, prompts, or internal behavior. It wraps API calls with pre-processing (policy injection) and post-processing (cryptographic locking).

Runtime Enforcement Flow

1
Request Ingestion

Enterprise application makes Claude API call through Bickford middleware. Request includes enforcement policy selection (FedRAMP, HIPAA, SEC, etc.)

2
Policy Validation

Bickford loads appropriate constraint policy. Validates user authorization, checks data access permissions, confirms regulatory compliance requirements.

3
Constitutional AI Invocation

Request forwarded to Claude API with policy context. Claude generates response using Constitutional AI training (inherent safety).

4
Constraint Enforcement

Claude response evaluated against deterministic policy rules. Bickford validates: no prohibited actions, no sensitive data leakage, compliance requirements met.

5
Audit Trail Generation

Complete decision pathway logged: input prompt (sanitized), constraints applied, response generated, validation results, timestamp, user context.

6
Cryptographic Lock

Audit record cryptographically hashed using SHA-256. Hash stored in tamper-evident ledger. Response returned with proof of enforcement.
